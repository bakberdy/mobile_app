# Add Injectable Module Entry

## When to use AppModule

Only classes that **cannot be annotated** with `@LazySingleton` / `@Injectable` directly belong in `AppModule`. These are:
- Third-party package classes (`SharedPreferences`, `Dio`, `FlutterSecureStorage`, etc.)
- Platform singletons that need constructor arguments from other registered deps
- Classes requiring async initialization

Everything else (your own repositories, use cases, BLoCs, data sources) uses annotations on the class itself.

---

## File to edit

```
lib/src/config/di/app_module.dart
```

`injection.config.dart` is **never edited manually** — it is regenerated by `build_runner`.

---

## Annotation decision

| Scenario | Annotation | Return type |
|----------|-----------|-------------|
| Sync, no deps | `@singleton` getter | `TypeName` |
| Sync, needs other deps | `@singleton` method | `TypeName` |
| Async init (e.g. `SharedPreferences`) | `@preResolve` + `@singleton` | `Future<TypeName>` |
| Lazy, only created on first use | `@lazySingleton` method | `TypeName` |
| Bind impl to interface | `@singleton` method returning interface type | `InterfaceType` |

Use `@singleton` (eager) in `AppModule` when the dep must be ready before the app starts. For everything else prefer `@lazySingleton`.

---

## Patterns

**Simple sync singleton (no deps):**
```dart
@singleton
MyService get myService => MyService();
```

**Singleton with injected deps:**
```dart
@singleton
MyClient myClient(AppConfig config, TokenStorage tokenStorage) =>
    MyClient(baseUrl: config.baseUrl, tokenStorage: tokenStorage);
```

**Async singleton (`@preResolve`):**
```dart
@preResolve
@singleton
Future<SharedPreferences> get sharedPreferences async =>
    await SharedPreferences.getInstance();
```
`@preResolve` tells injectable to `await` this before the container is ready. Only use it when the dep is required at startup.

**Bind impl to interface:**
```dart
@singleton
TokenStorage tokenStorage(FlutterSecureStorage secureStorage) =>
    TokenStorageImpl(secureStorage);
```
The return type is the **interface** (`TokenStorage`), not the impl — this is how injectable resolves interface injections.

**Lazy singleton (created on first use):**
```dart
@lazySingleton
ApiClient apiClient(TokenStorage tokenStorage, AppConfig appConfig) =>
    ApiClient(
      ApiConfig(baseUrl: appConfig.baseUrl),
      tokenStorage,
    );
```

## Full example adding a new entry

```dart
@module
abstract class AppModule {
  // ... existing entries ...

  @singleton
  MyAnalyticsClient analyticsClient(AppConfig config) =>
      MyAnalyticsClient(apiKey: config.analyticsKey);
}
```

Import the package at the top of `app_module.dart`. Do not import it anywhere else in the app — inject `MyAnalyticsClient` via constructor.

---

## After editing AppModule

```bash
dart run build_runner build -d
```

Verify the new type appears in `lib/src/config/di/injection.config.dart`.

---

## Rules

- ❌ Never edit `injection.config.dart` manually
- ❌ Never call `GetIt.instance.registerSingleton(...)` or `sl.registerFactory(...)` inline
- ❌ Never call `sl<T>()` inside a class body — inject via constructor only
- ✅ Return the **interface type** (not the impl) when binding impl to interface
- ✅ Add `@preResolve` only for deps needed before app start (async init)
- ✅ Prefer `@lazySingleton` over `@singleton` unless eager init is required
