# Add Route

## Files to edit

| File | Change |
|------|--------|
| Screen file | Ensure `@RoutePage()` is on the wrapper class |
| `lib/src/config/router/app_router.dart` | Import screen + add `AutoRoute(...)` |

Generated file `app_router.gr.dart` is **never edited manually** ŌĆö it is regenerated by `build_runner`.

---

## Step 1 ŌĆö Verify `@RoutePage()` on the screen

The screen's **wrapper** class (the public one, not `_Content`) must have the annotation:

```dart
import 'package:auto_route/auto_route.dart';

@RoutePage()
class BookingListScreen extends StatelessWidget { ... }
```

- ŌØī Missing annotation ŌåÆ AutoRoute won't generate a route class
- ŌØī On `_BookingListScreenContent` ŌåÆ wrong widget, content is private

---

## Step 2 ŌĆö Import the screen in `app_router.dart`

Add the import at the top of `lib/src/config/router/app_router.dart`:

```dart
import 'package:<package>/src/features/<feature>/presentation/screens/<feature>_<name>_screen.dart';
```

---

## Step 3 ŌĆö Declare the route

Add an `AutoRoute` entry inside the `routes` list in `app_router.dart`.

**Top-level screen (modal, standalone):**
```dart
AutoRoute(page: BookingListRoute.page, path: '/booking'),
```

**Top-level with path parameter:**
```dart
AutoRoute(page: BookingDetailRoute.page, path: '/booking/:id'),
```

**Child of a tab/shell route:**
```dart
AutoRoute(
  page: MainNavigationRoute.page,
  guards: [_authGuard],
  children: [
    AutoRoute(page: HomeRoute.page, path: 'home', initial: true),
    AutoRoute(page: BookingListRoute.page, path: 'booking'),  // ŌåÉ add here
  ],
),
```

**Initial route** (first screen shown, only one allowed):
```dart
AutoRoute(page: OnboardingRoute.page, path: '/', initial: true),
```

Route name convention: `BookingListScreen` ŌåÆ `BookingListRoute`
(`replaceInRouteName: 'Screen,Route'` is already configured in `AppRouter`).

---

## Step 4 ŌĆö Run build_runner

```bash
dart run build_runner build -d
```

Verify `lib/src/config/router/app_router.gr.dart` now contains a `BookingListRoute` class.

## Route parameters (when the screen takes params)

Declare `@PathParam` or `@QueryParam` on the screen constructor ŌĆö not via widget constructor:

```dart
// Screen
@RoutePage()
class BookingDetailScreen extends StatelessWidget {
  final String bookingId;
  const BookingDetailScreen({@PathParam('id') required this.bookingId, super.key});
}

// Router
AutoRoute(page: BookingDetailRoute.page, path: '/booking/:id'),

// Usage
context.router.push(BookingDetailRoute(bookingId: booking.id));
```

Query param variant:
```dart
final String? filter;
const BookingListScreen({@QueryParam('filter') this.filter, super.key});
// Usage: context.router.push(BookingListRoute(filter: 'upcoming'));
```

---

## Navigation API (reference)

```dart
context.router.push(const BookingListRoute());           // push
context.router.replace(const BookingListRoute());        // replace current
context.router.pop();                                    // back
context.router.replaceAll([const MainNavigationRoute()]); // reset stack
```

- ŌØī Never use `Navigator.of(context).push(...)`
- ŌØī Never navigate inside a BLoC or UseCase ŌĆö only in `BlocListener`

## Checklist

- [ ] Screen wrapper has `@RoutePage()` annotation
- [ ] Screen file imported in `app_router.dart`
- [ ] `AutoRoute(page: XxxRoute.page, ...)` added in the correct position (top-level or child)
- [ ] Path set correctly ŌĆö path params use `:paramName` syntax
- [ ] `initial: true` only if this is the app's first screen
- [ ] `guards: [_authGuard]` added if the route needs auth protection
- [ ] `dart run build_runner build -d` ran successfully
- [ ] `XxxRoute` class visible in `app_router.gr.dart`
